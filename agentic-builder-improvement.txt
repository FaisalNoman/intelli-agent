  ### What agentic-builder actually is (your baseline)                          
                                                                                
  Solid foundation. In-session SDLC orchestrator for Claude Code — no separate  
  API key, no external process. Key differentiators: global dependency-graph    
  scheduler that keeps agents saturated (no sprint barriers), three             
  auto-detected modes (GREENFIELD / FEATURE / SURGICAL), TDD discipline baked   
  in, design-system routing with wireframe approval, and a live dashboard with  
  interactive approval UI.                                                      
                                                                                
  **The honest weaknesses before we compare:**                                  
  - 2 stars, 0 forks. You're essentially pre-adoption.                          
  - No persistent memory across sessions. Every run starts cold.                
  - No cross-session learning. It doesn't get smarter.                          
  - No budget/cost tracking.                                                    
  - No file conflict detection between parallel agents.                         
  - No session replay or audit trail.                                           
  - No CI/CD integration or headless mode.                                      
  - Single-harness only (Claude Code). No multi-provider routing.

  ### Competitor breakdown                                                      
                                                                                
  **Superpowers (233k stars, 609 commits, v6.0.3 — this is the benchmark)**     
                                                                                
  What it does better than you: multi-harness support (Claude Code, Codex,      
  Cursor, Gemini, Kimi, Pi, OpenCode), a complete eval system with the          
  `writing-skills` skill for self-improving the framework, systematic-debugging 
  as a dedicated skill, and `using-git-worktrees` as a first-class workflow.    
  The philosophy is "mandatory workflows, not suggestions" — skills trigger     
  automatically.                                                                
                                                                                
  What you do better: your live dashboard with interactive approvals is         
  genuinely ahead. Superpowers has no equivalent real-time UI. Your             
  dependency-graph scheduler is also more explicit and visual.                  
                                                                                
  **What to borrow:** the `systematic-debugging` 4-phase root-cause skill, the  
  `verification-before-completion` discipline, and the `receiving-code-review`  
  skill. Also: their approach to `finishing-a-development-branch` (merge/PR     
  decision workflow) — you have no equivalent gate for branch lifecycle.        
                                                                                
  ---                                                                           
                                                                                
  **Ruflo (61.5k stars, 1,546 releases, active enterprise-grade project)**      
                                                                                
  This is a different category. Ruflo is a full meta-harness: 100+ agents,      
  self-learning via SONA neural patterns, HNSW vector memory that persists and  
  improves across sessions, agent federation across machines with zero-trust    
  mTLS, GOAP A* planning at goal.ruv.io, and a plugin marketplace with 35+      
  plugins. It also has multi-provider routing (Claude, GPT, Gemini, Ollama with 
  failover), 12 background workers, cost tracking, security auditing, and a     
  web UI.                                                                       
                                                                                
  What you do better: simpler to install, zero credentials, tighter             
  SDLC-specific workflow. Ruflo's complexity is also its weakness — the "you    
  don't need to learn 314 MCP tools" disclaimer in their own README is a red    
  flag about cognitive load.                                                    
                                                                                
  **What to borrow:** persistent session memory (even a lightweight version),   
  cost/token tracking per agent run, session recording + replay (JSONL event    
  log), and their `ruflo-security-audit` pattern for CVE scanning as a          
  post-build gate.

  2026)**

  right: explicit budget enforcement with hard cost limits, file conflict       
  detection (pessimistic locking), smart model tiering (Opus for                
  planning/review, Haiku for workers = 3x cheaper), and a dedicated quality     
  gate phase where Opus reviews combined agent output. Session replay via JSONL

  What you do better: your dependency-graph is more sophisticated, your

  current setup has no protection against agents stepping on the same file),    
  budget hard limits, and the tiered model selection pattern (you're presumably 
  using the same model everywhere — that's wasteful).

  ### Prioritized feature additions (ranked by impact vs effort)                
                                                                                
  **High impact, manageable effort:**                                           
                                                                                
  1. **File conflict detection / pessimistic locking** — steal directly from    
  claude-swarm. Without this, parallel agents will corrupt each other's work on 
  any real codebase. This is a correctness bug, not a feature gap.              
                                                                                
  2. **Budget + cost tracking per run** — Ruflo and claude-swarm both do this.  
  Users will want to know what a build costs before they let it run for 2       
  hours.                                                                        
                                                                                
  3. **Session recording + replay (JSONL)** — lightweight to implement, high    
  value for debugging failed runs and auditing agent behavior.                  
                                                                                
  4. **Systematic debugging skill** — borrow from Superpowers. A dedicated      
  4-phase root-cause skill would complement your SURGICAL mode.                 
                                                                                
  **Medium impact, more work:**                                                 
                                                                                
  5. **Persistent cross-session memory** — even a simple project-scoped         
  `.agentic-builder/memory.json` would be meaningful. Full HNSW like Ruflo is   
  overkill for now, but storing what was built, what failed, and what patterns  
  worked gives the next run a head start.                                       
                                                                                
  6. **Branch lifecycle gate** — `finishing-a-development-branch` equivalent.   
  Right now you have no explicit workflow for when the build is done: merge?    
  PR? discard? squash? That's a missing SDLC phase.                             
                                                                                
  7. **Tiered model routing** — use a cheaper model (Haiku) for worker agents,  
  Sonnet/Opus for planning and review. claude-swarm's architecture here is      
  sound. Cuts cost by 3x on large runs.                                         
                                                                                
  **Lower priority but worth tracking:**                                        
                                                                                
  8. **Multi-harness support** — Superpowers targets Claude Code, Cursor,       
  Codex, Gemini, Pi etc. You're Claude Code only. This limits your TAM but      
  isn't critical until you have adoption.                                       
                                                                                
  9. **CI/CD / headless mode** — you mention "Engine B" in your notes but it's  
  not shipped. Until it is, you can't run in pipelines.                         
                                                                                
  10. **Self-learning** — Ruflo's SONA is architecturally ambitious. Don't try  
  to replicate it now. But a simple "what worked last time in this project"     
  retrieval would deliver 80% of the value.

  ---
  P0 — Correctness: runtime file-ownership guard

  Gap: conflict handled only at design time (phases.md:89 "split if two tasks
  touch same file"). If decomposition misjudges → parallel agents clobber. No
  runtime guard.
  Build:
  - New references/file-ownership.md. Each task in *-tasks.json declares writes:
  [globs].
  - Scheduler (scheduler.md dispatch loop) — before filling slots, skip any
  ready node whose writes glob-overlaps an in-flight node's writes; keep it
  queued. Pure scheduling constraint, no OS locks.
  - State: plan/state/locks.json = active write-claims for dashboard viz.
  Effort: M. Accept: two tasks writing same path never run concurrently; proven
  by a conflict-injection test.

  P1 — Cost control (token measure already done)

  Gap: no model tiering (all agents same model), no budget cap.
  Build:
  - agent-contracts.md + registry: add model per agent role — Haiku for
  impl/worker, Sonnet/Opus for architect/review/plan. Pass via Agent tool model
  param.
  - references/budget.md + Stage 0 preflight asks soft max_tokens / max_usd.
  Scheduler checks live measured total (already have token-report.mjs) each
  loop; on breach → pause, write approval prompt to dashboard (reuse existing
  modal).
  Effort: S (tiering) + M (budget). Accept: large run shows ~3x worker cost
  drop; breach pauses for approval.

  P2 — Cross-session memory

  Gap: cache is per-run only; every build starts cold.
  Build:
  - .agentic-builder/memory.json at project root (survives runs): records per
  milestone — what built, what failed + fix, decisions, file map.
  - Stage 0 loads it → injects "prior-run context" into architect/plan agents.
  - No HNSW. Plain JSON + keyword filter. Optional chromadb if caps.memory.
  Effort: M. Accept: 2nd run on same project references prior decisions, skips
  re-derivation.

  P3 — Session replay / audit

  Gap: transcript exists but no run-scoped event log / replay.
  Build: append-only plan/state/events.jsonl (agent start/done/blocked, gate,
  approval, cost tick). Dashboard "Replay" tab scrubs it. Cheap — orchestrator
  already has these transition points.
  Effort: S–M. Accept: failed run reconstructable from JSONL.
